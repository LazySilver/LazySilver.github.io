问题汇总

发表于2025-10-28|更新于2025-10-28
|浏览量:

  1. hexo new “文件名”后，tags： 后面一定要==跟一个空格==，不然会报错（基本所有都要跟空格）

  2. 我的markdown的高亮是====，但是hexo好像显示不了高亮
    解决办法：（hexo自己不带高亮的渲染器，需要插件）

    1. ==卸载可能冲突的插件==
      npm uninstall markdown-it-emphasis-alt --save

    2. ==安装markdown-it-mark插件==
      npm install hexo-renderer-markdown-it markdown-it-mark --save

    3. ==在hexo根目录的_config.yml最后加上以下代码==

    4. hexo cl

    5. hexo g

    6. hexo s测试

    7. hexo d推送

  3. vscode的ctrl+shift+P可以搜索设置

  4. vscode的路径不能有==中文==，命名不能有==中文==

  5. git推送时，如果有东西没法推，一般是某个文件被视为了==子仓库==，而且有.git文件夹，建议把主仓库和子仓库的.git==全删了==，然后检查.gitignore文件，确保没漏再推送一次

  6. vscode按==ctrl+f==可以==搜索==指定关键字
